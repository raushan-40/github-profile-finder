# Prompts Used During Development

This document contains the prompts used while developing the **GitHub Profile Finder** project with the assistance of GitHub Copilot and ChatGPT.

---

## Prompt 1 – Project Setup

```
Create a GitHub Profile Finder using HTML, CSS, and Vanilla JavaScript. Keep the project beginner-friendly, responsive, and well-structured. Do not use any frameworks.
```

---

## Prompt 2 – HTML Structure

```
Create the HTML structure for a GitHub Profile Finder with a search input, search button, profile section, and repositories section using semantic HTML.
```

---

## Prompt 3 – Modern UI

```
Design a modern dark-themed responsive UI using CSS. Use cards, proper spacing, rounded corners, and hover effects while keeping the design clean and professional.
```

---

## Prompt 4 – Search Functionality

```
Implement JavaScript to read the GitHub username from the input field and trigger the search when the user clicks the Search button.
```

---

## Prompt 5 – GitHub API Integration

```
Fetch GitHub user data using the Fetch API and async/await. Parse the JSON response and return the user information.
```

---

## Prompt 6 – Display Profile

```
Render the GitHub user's avatar, name, username, bio, join date, portfolio link, followers, following, and public repositories in the UI.
```

---

## Prompt 7 – Loading State

```
Display a loading spinner while the API request is in progress and remove it after the request completes.
```

---

## Prompt 8 – Error Handling

```
Handle invalid usernames gracefully by displaying a user-friendly 'User Not Found' message instead of crashing the application.
```

---

## Prompt 9 – Fetch Repositories

```
Fetch the user's repositories using the GitHub REST API and display only the latest five repositories with clickable links.
```

---

## Prompt 10 – Date Formatting

```
Convert GitHub ISO date strings into a human-readable format such as '12 Apr 2025'.
```

---

## Prompt 11 – UI Improvements

```
Improve spacing, typography, hover effects, responsiveness, and overall user experience without changing the existing functionality.
```

---

## Prompt 12 – Bug Fix

```
Find and fix the issue where clicking a repository opens two browser tabs. Ensure each repository opens only one new tab.
```

---

## Prompt 13 – Code Refactoring

```
Refactor the JavaScript into small reusable functions while keeping the code easy to understand and beginner-friendly.
```

---

## Prompt 14 – Final Review

```
Review the entire project for responsiveness, accessibility, code quality, UI consistency, and JavaScript best practices. Make only necessary improvements without changing the project functionality.
```

---

# AI Tools Used

- GitHub Copilot
- ChatGPT

# Project

**GitHub Profile Finder**
